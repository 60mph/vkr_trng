# Схема №7 — Дрожание тактовых сигналов

## BOM

Только Arduino Nano. Ни одного внешнего компонента.

## Принцип

ATmega328P даёт нам "из коробки" два тактовых источника:

- **TIMER1** работает от внешнего кварца 16 МГц (низкий фазовый шум,
  σ_jitter < 50 пс).
- **WDT** работает от **внутреннего RC-генератора 128 кГц** — он сильно
  шумнее, σ_jitter ≈ 5–10 мкс при комнатной температуре.

Прошивка `07_clock_jitter` каждый раз, когда срабатывает WDT-прерывание
(~раз в 16 мс), читает значение TIMER1. Младшие биты этого значения
несут информацию **только** о случайной фазе RC-генератора WDT — они
непредсказуемы.

## Подключение

Никакого. Arduino работает на USB-питании. Желательно стабильное USB
(не на хабах с дешёвыми БП, чтобы пульсация по +5В не модулировала RC).

## Проверка

После прошивки в Serial Monitor (1 Мбод) должны идти 16-битные значения,
каждый раз разные в младших разрядах. Если значения почти не меняются —
WDT не запустился, проверьте включение WDIE в `wdt_setup()`.
