# Подсистема ГИСП на Arduino — практическая часть дипломной работы

Каталог `trng/` содержит **полный практический фреймворк** для построения нескольких
физических генераторов истинно случайных последовательностей (ГИСП) на базе Arduino Nano,
сбора энтропии, постобработки, статистического анализа и тестирования NIST STS.

Архитектура спроектирована так, чтобы каждый источник энтропии был **изолирован**
(своя прошивка + своя схема + свой файл результатов), но проходил через **единый
конвейер обработки**. Это позволяет:

- проводить **сравнительный анализ** разных источников при одинаковых условиях;
- легко **добавлять новые источники** без переписывания инфраструктуры;
- **импортировать результаты** в дипломную работу (`parts/05-chapter2.tex` и
  `parts/06-chapter3.tex`) в виде готовых markdown-фрагментов и графиков из
  `trng/data/reports/`.

## Список реализуемых источников энтропии

Источники подобраны под имеющийся набор компонентов. Каждый из них упомянут
в дипломной работе (главы 2.1–2.3), что позволяет привязать практические
результаты к теоретическим разделам.

| № | Источник                                    | Раздел дипломной работы | Компоненты                                     |
|---|---------------------------------------------|-----------------|------------------------------------------------|
| 1 | Тепловой шум (Johnson–Nyquist)              | 2.1 (тепловой)  | резистор 1 МОм + LM358 + Arduino A0            |
| 2 | Шум стабилитрона (лавинный/Зенера)          | 2.1 (лавинный)  | 5,1 В стабилитрон + LM358 + Arduino A0         |
| 3 | Шум BE-перехода BJT (дробовой/лавинный)     | 2.1 (дробовой)  | 2N3904 (BE reverse) + LM358 + Arduino A0       |
| 4 | Акустический шум                            | 2.3             | KY-037 → Arduino A0                            |
| 5 | MEMS-шум акселерометра/гироскопа            | 2.3             | MPU-6050 (I²C) → Arduino                       |
| 6 | Электромагнитные наводки на плавающий ADC   | 2.1 / 2.3       | только Arduino (A0 не подключен)               |
| 7 | Дрожание тактовых сигналов (clock jitter)   | 2.1             | только Arduino (WDT vs кварц)                  |
| 8 | Гибридный (XOR нескольких источников)       | 2.4 / 3.1       | комбинация 1–7 + SHA-256 на ПК                 |

## Быстрый старт

### 1. Зависимости (Python 3.9+)

```bash
cd trng
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Сборка NIST STS

```bash
cat nist_sts/README.md   # инструкция по сборке sts-2.1.2 из исходников
```

### 3. Сборка устройства

Откройте `hardware/0X_*.md` для нужного источника и соберите схему на макетной
плате MB-102. Подключите Arduino Nano по USB-C.

### 4. Прошивка Arduino

Откройте `firmware/0X_*/0X_*.ino` в Arduino IDE (≥ 2.0) или `arduino-cli`,
выберите плату Arduino Nano (ATmega328P, Old Bootloader при необходимости),
скомпилируйте и загрузите.

### 5. Захват данных и end-to-end анализ

```bash
# Полный конвейер: захват 10 МБ, постобработка, анализ, NIST STS
python scripts/pipeline.py --source 02_zener --port /dev/ttyUSB0 --bytes 10485760
```

Результат окажется в `data/reports/02_zener/`:

- `report.md`      — сводка с p-value по NIST STS, Shannon, min-entropy;
- `bits_image.png` — визуализация итогового битового потока (512×512 по умолчанию);
- `histogram.png`  — гистограмма выборок;
- `autocorr.png`   — автокорреляция;
- `psd.png`        — спектральная плотность мощности;
- `nist.csv`       — таблица результатов всех 15 тестов NIST STS.

### 6. Импорт в дипломную работу

Содержимое `docs/0X_*.md` написано так, что включается в LaTeX через `pandoc`
или вставляется фрагментами в `parts/05-chapter2.tex`. Картинки можно скопировать
в `pandoc_media/media/` и подключить через `\includegraphics`.

## Принципы дизайна

1. **Единый бинарный протокол** (`firmware/common/trng_protocol.h`) —
   любая прошивка отдаёт по USB-CDC одинаковый поток `uint16_le` отсчётов
   после короткого ASCII-баннера с метаданными.

2. **Пер-источниковый каталог** в `firmware/`, `hardware/`, `docs/` — добавление
   нового источника = новая папка, инфраструктура не меняется.

3. **Идемпотентность**: все Python-инструменты принимают входной файл и пишут
   новый файл без перезаписи исходных данных.

4. **Воспроизводимость**: каждый отчёт включает временную метку, версию
   прошивки, частоту дискретизации, метод постобработки, чтобы результаты
   из дипломной работы можно было перепроверить.

5. **Разделение «сырых» и «постобработанных» данных**: NIST STS прогоняется
   как до, так и после `von_neumann.py` / `mix.py`, чтобы продемонстрировать
   эффект постобработки (важно для главы 2.4 дипломной работы).

## Что писать в дипломную работу

После того, как для источника собраны данные и сгенерирован отчёт, материал
для дипломной работы формируется так:

1. Из `hardware/0X_*.md` — схема и описание установки → новый подраздел в
   главе 2.
2. Из `data/reports/0X_*/report.md` — таблица NIST STS, оценки энтропии,
   графики → подраздел "Результаты".
3. Из `docs/0X_*.md` — обсуждение, преимущества/недостатки, сравнение с
   теорией → выводы по подразделу.
4. В главу 3 («Сравнительная характеристика») добавляется итоговая таблица из
   `data/reports/comparison.md`, генерируемая `analysis/report.py --compare`.
