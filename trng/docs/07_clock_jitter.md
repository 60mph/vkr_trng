# Источник №7 — Дрожание тактовых сигналов (clock jitter)

**Раздел дипломной работы:** 2.1 (фундаментальные шумы) + общая идея "комбинации
двух осцилляторов разной природы".

## Физический принцип

ATmega328P имеет два независимых тактовых источника:

- **Кварцевый резонатор 16 МГц** — фазовый шум очень низкий, σ_jitter < 50 пс.
- **Внутренний RC-генератор WDT** ≈ 128 кГц — фазовый шум значительный
  (σ_jitter ≈ единицы микросекунд), сильно зависит от температуры и питания.

Прошивка `07_clock_jitter` считает, сколько тиков 16 МГц прошло за один
"тик" WDT (~16 мс ≈ 256 000 тактов). Младшие биты этого числа меняются
случайно от опроса к опросу — это и есть энтропия.

Скорость генерации: ~60 опросов/с × ~6 младших бит ≈ **360 бит/с**. Очень
медленно, но **не требует никакой внешней электроники** — поэтому идеально
для bring-up'а тестового стенда.

| Температура                     |                |

Аппаратная часть — `hardware/07_clock_jitter.md` (без внешних деталей).

## Команда запуска

```bash
python scripts/pipeline.py --source 07_clock_jitter --port /dev/ttyUSB0 \
    --bytes 65536 --bits 6 --sts ./nist_sts/sts-2.1.2
```

(Из-за низкой скорости берём всего 64 КБ; хватит для NIST STS на 1 поток
по 1 М бит.)

## Результаты (заполняются после прогона)

- Ожидается, что прямые LSB **проходят NIST STS** уже без VN — RC-генератор
  даёт качественный шум, главная проблема — медленная скорость, а не качество.
