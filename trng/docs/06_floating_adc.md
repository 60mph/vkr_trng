# Источник №6 — Шум плавающего ADC-пина

**Раздел дипломной работы:** 2.1 (тепловой / 1/f) + 2.3 (электромагнитные наводки),
обсуждается также в главе 3 как "хаотический источник" с низким доверием.

## Физический принцип

Плавающий пин ADC накапливает заряд на S/H-конденсаторе, который определяется:

- утечкой через защитные диоды;
- наводками от внешних ЭМ-полей (50/60 Гц, Wi-Fi, USB);
- тепловым шумом самого S/H;
- историческим состоянием предыдущей выборки.

Это очень слабый и сильно коррелированный источник. **Без постобработки**
выборки практически не проходят NIST. Однако служит ценным baseline'ом — в
дипломной работе уместно показать, что простой "софтовый" подход даёт плохие
результаты по сравнению с настоящей аналоговой схемой.

Аппаратная часть — `hardware/06_floating_adc.md`.

## Команда запуска

```bash
python scripts/pipeline.py --source 06_floating_adc --port /dev/ttyUSB0 \
    --bytes 10485760 --bits 1 --sts ./nist_sts/sts-2.1.2
```

## Что обычно выходит

- На PSD — **острый пик 50 Гц** (сетевая наводка).
- В гистограмме — несколько узких пиков (S/H "залипает" на старом значении).
- NIST STS — большая часть тестов **FAIL** на сырых LSB.
- После `mix.py --mode sha256` — **PASS** (это и есть демонстрация важности
  криптографического extractor'а из раздела 2.4 дипломной работы).
