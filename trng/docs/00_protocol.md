# Общий протокол передачи данных Arduino → ПК

Все 8 источников энтропии используют единый бинарный USB-CDC-протокол,
определённый в `firmware/common/trng_protocol.h`. Это критично для того,
чтобы инфраструктура Python-инструментов (`capture/`, `analysis/`) работала
без изменений с любым источником.

## Поток

```
        ┌─────────────────────────────────────────────┐
RESET → │ ASCII-метаданные                            │
        │ # TRNG_SOURCE=02_zener                      │
        │ # FIRMWARE_VERSION=1.0                      │
        │ # SAMPLE_FORMAT=u16le                       │
        │ # SAMPLE_BITS=10                            │
        │ # SAMPLE_RATE_HZ=76923                      │
        │ # ADC_REF=AVCC                              │
        │ # NOTES=...                                 │
        │ BEGIN\n                                     │
        ├─────────────────────────────────────────────┤
        │ uint16_le, uint16_le, ... (бесконечно)      │
        └─────────────────────────────────────────────┘
```

## Скорость

UART настроен на 1 000 000 бит/с (1 Мбод). Это даёт максимальную полезную
пропускную способность ≈ 100 КиБ/с (= 50 000 uint16-отсчётов/с). Для большинства
источников этого достаточно с запасом, ADC ATmega328P даёт ≤ 76 квыб/с в
free-running режиме.

## Метаданные

Каждое поле — пара `# КЛЮЧ=ЗНАЧЕНИЕ`. Скрипт `capture/capture_serial.py` парсит
все поля до строки `BEGIN` и сохраняет их в `<run>.meta` (JSON).

| Ключ            | Назначение                                          |
|-----------------|-----------------------------------------------------|
| TRNG_SOURCE     | идентификатор источника (`02_zener` и т.п.)         |
| FIRMWARE_VERSION| версия прошивки                                     |
| SAMPLE_FORMAT   | формат отсчёта; сейчас всегда `u16le`               |
| SAMPLE_BITS     | сколько бит в отсчёте реально несут информацию      |
| SAMPLE_RATE_HZ  | частота дискретизации, нужна для PSD                |
| ADC_REF         | какое опорное использовалось (AVCC/INTERNAL_1V1/…)  |
| NOTES           | произвольный комментарий                            |

## Расширение

Если в будущем потребуется отдельный бинарный фрейминг (CRC, длина пакета —
например, для приёмника, который может терять биты), его легко добавить в
`trng_protocol.h` без изменения прошивок (через новое значение `SAMPLE_FORMAT`)
и в `capture_serial.py`.
