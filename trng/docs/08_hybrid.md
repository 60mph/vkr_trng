# Источник №8 — Гибридный TRNG

**Разделы дипломной работы:** 2.4 (постобработка, XOR-корректоры) + 3.2 ("гибридные
многоканальные схемы").

## Идея

Объединить **независимые** физические источники, чтобы:

1. компенсировать слабые стороны каждого (детерминизм, помехи 50 Гц, дрейф);
2. получить итоговый поток с **min-entropy ≥ сумме компонентов** — это
   следует из того, что XOR независимых битовых источников b1⊕b2⊕…⊕bk
   близок к максимально энтропийному, если хотя бы один из них имеет высокое
   качество.

Реализованы **два уровня** комбинирования:

### Уровень 1 — на Arduino (`firmware/08_hybrid.ino`)

В реальном времени XOR'им байт за байтом:

- ADC от внешнего источника (A0) — обычно 02_zener или 03_bjt_avalanche;
- ADC от плавающего пина (A1) — наводки/тепловой шум S/H;
- 6 LSB MPU-6050 акселерометра/гироскопа.

Скорость ≈ 5 кБ/с. Соответствует "XOR-корректору" из раздела 2.4 дипломной работы.

### Уровень 2 — на ПК (`capture/mix.py`)

После записи каждого источника отдельно — комбинируем сразу несколько
файлов через SHA-256 поверх потоков. Это **криптографически стойкий
extractor** в духе SP 800-90B/C:

```bash
python capture/mix.py --mode sha256 \
    --in data/processed/02_zener/run_001_lsb1_vn.bin \
    --in data/processed/04_microphone/run_001_sha.bin \
    --in data/processed/05_mpu6050/run_001_lsb6.bin \
    --out data/processed/08_hybrid/sha_mix.bin
```

## Установка эксперимента

Для уровня 1 (Arduino) — нужны одновременно:

- TL072 + источник на A0 (например, 02_zener);
- MPU-6050 на I²C;
- свободный пин A1.

Для уровня 2 (ПК) — подойдут уже снятые данные с любых источников.

| Параметр                | Значение  |
|-------------------------|-----------|
| Дата                    |           |

| Длительность, мин       |           |

## Команда запуска (уровень 1)

```bash
python scripts/pipeline.py --source 08_hybrid --port /dev/ttyUSB0 \
    --bytes 10485760 --bits 8 --no-vn --sts ./nist_sts/sts-2.1.2
```

(Уровень 1 уже отдаёт 8-битные XOR-байты, дополнительный extract_bits
не нужен — поэтому `--bits 8 --no-vn`.)

## Команда запуска (уровень 2)

См. пример выше + `analysis/report.py --source 08_hybrid_sha`.

## Ожидания

- Уровень 1 (Arduino XOR) — большинство NIST STS PASS, при условии что
  один из источников действительно высококачественный.
- Уровень 2 (SHA-256 extractor) — ВСЕ NIST STS PASS, даже если на входе
  плохие данные. Это подтверждает теорему о "криптографической нормализации".
